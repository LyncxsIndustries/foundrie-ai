# 30 - ZIP Builder

## Goal

Build server-side ZIP assembly with JSZip.

## Context To Read First

- `context/project-overview.md`
- `context/architecture-context.md`
- `context/ui-context.md`
- `context/code-standards.md`
- `context/ai-workflow-rules.md`
- `context/progress-tracker.md`

## Context7 Docs To Check

- JSZip `/stuk/jszip`

Use installed Context7 skills or:

```bash
npx ctx7 library <library> "<specific question>"
npx ctx7 docs <libraryId> "<specific question>"
```

## Implementation

- Create ZIP root folder named `{project-slug}_{timestamp}`.
- Add root `AGENTS.md`.
- Add root `ARTKINS_STYLE_GUIDE.md`.
- Add `.agents/skills/` when project-specific skills exist.
- Add `context/` files.
- Add `feature-specs/` files in numeric order.
- Add `research/` folder:
  - `PROJECT_RESEARCH.md`
  - additional research Markdown files
  - populated research asset subfolders only
- Add `diagrams/` grouped by category.
- Add `requirements/` docs.
- Use error placeholders for missing diagram PNGs.
- Use `dbRead` for ZIP assembly reads when a read replica is configured.
- Use a consistent snapshot for multi-table ZIP reads, preferably a `RepeatableRead` transaction where supported by the chosen Prisma setup.
- Select only fields needed for ZIP assembly.
- Do not fetch large React Flow JSON unless the ZIP explicitly needs source diagram JSON.
- Use ordered indexed reads: context by file type, specs by order, diagrams by category/order.
- Fetch research documents ordered by creation/title and research assets grouped by type.
- Download Blob-backed research assets into the matching `research/` subfolder.
- Use text placeholders when an external research asset cannot be included, preserving source URL and error reason.

## Scope Limits

- Do not implement later feature specs early.
- Do not introduce undocumented architecture changes.
- Do not bypass the storage, auth, AI, or Context7 rules in the context files.

## Check When Done

- The feature works within its defined scope.
- Relevant library docs were checked with Context7.
- Types are strict and external input is validated.
- Access control is enforced where data is read or mutated.
- ZIP includes `research/PROJECT_RESEARCH.md`.
- ZIP includes root `ARTKINS_STYLE_GUIDE.md`.
- ZIP includes uploaded/captured research assets in stable subfolders.
- ZIP includes `.agents/skills/` when generated by Feature 27.
- Feature specs and AGENTS.md can reference research paths that exist in the ZIP.
- `context/progress-tracker.md` is updated.
- `npm run build` passes once application code exists.
