# Feature 35 - Project Member Schema

## Type

NEW FEATURE

## What This Delivers

Adds the `ProjectMember` model and `ProjectMemberRole` enum to the Prisma schema, establishing the data foundation for the Owner + Collaborator authorization model. When a user creates a project, a `ProjectMember` row with role `OWNER` is automatically implied by the existing `Project.userId` field. This spec adds the table for explicit membership tracking so Collaborators can be invited in later specs.

- Add `ProjectMemberRole` enum with values `OWNER` and `COLLABORATOR`.
- Add `ProjectMember` model with fields: `id`, `projectId`, `userId`, `role` (ProjectMemberRole), `invitedByUserId` (optional String), `joinedAt` (DateTime), `createdAt`, `updatedAt`.
- Add relations: `ProjectMember` belongs to `Project` and `User`.
- Add `ProjectMember[]` relation on `Project` model.
- Add `ProjectMember[]` relation on `User` model (as `memberships`).
- Add `@@unique([projectId, userId])` to prevent duplicate memberships.
- Add `@@index([userId])` for "shared with me" queries.
- Add `@@index([projectId, role])` for role-filtered queries.
